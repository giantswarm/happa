# Adding a new feature flag

Feature flags can be used for enabling or disabling specific features on specific installations. You work with them by changing a `ConfigMap` value to `true` or `false`.

### How to add a new feature flag?

Let's say we want to add a feature that enables or disables login. Let's start from the outside.

1. Add the default value into the Helm `ConfigMap` template.

This is so we could have a way to configure it.

<FileBlock fileName='helm/happa/templates/configmap.yaml'>
  {`apiVersion: v1
kind: ConfigMap
...
data:
  ...
+  feature-login: 'FALSE'
  ...
  `}
</FileBlock>

2. Map that value to an environmental variable in the Helm `Deployment` template.

This is so the container would know about our flag.

<FileBlock fileName='helm/happa/templates/deployment.yaml'>
  {`apiVersion: apps/v1
kind: Deployment
...
spec:
  ...
  template:
    ...
    spec:
      ...
      containers:
      - name: happa
        ...
        env:
        ...
+        - name: FEATURE_LOGIN
+          valueFrom:
+          configMapKeyRef:
+            name: happa-configmap
+            key: feature-login
        ...
`}
</FileBlock>

3. Add it to the window object in the index file template.

This is where the value gets into our code.

<FileBlock fileName='src/index.ejs'>
  {`<!DOCTYPE html>
<html>
    ...
    <script>
        ...
        window.featureFlags = {
                ...
+            FEATURE_LOGIN: <%= FEATURE_LOGIN %>,
                ...
        };
        ...
`}
</FileBlock>

4. Add it to the container startup script.

This is so the correct value arrives in the index file when the container starts.

<FileBlock fileName='scripts/start.sh'>
  {`#!/bin/sh
...
+ if [ "$FEATURE_LOGIN" = "TRUE" ]; then
+   sed -i "s|FEATURE_LOGIN: .\*|FEATURE_LOGIN: true,|" /www/index.html
+ else
+   sed -i "s|FEATURE_LOGIN: .\*|FEATURE_LOGIN: false,|" /www/index.html
+ fi
  ...`}
</FileBlock>

5. Define the new variable type in the window type definitions.

This is so TypeScript would know about our new flag.

<FileBlock fileName='src/@types/global.d.ts'>
  {`...
interface IFeatureFlags {
    ...
+    FEATURE_LOGIN: boolean;
    ...
}
...`}
</FileBlock>

6. Add it in the internal feature flags data structure.

This what you'll use in your code.

Add the new flag name to the `Feature` type union.

<FileBlock fileName='src/shared/featureFlags.ts'>
  {`...
export type Feature =
    ...
    | 'Login';
...`}
</FileBlock>

Add the flag's configuration.

<FileBlock fileName='src/shared/featureFlags.ts'>
  {`...
export const flags: Record<Feature, IFeatureFlag> = {
+    Login: {
         // This is the default value of the flag.
+        enabled: false,
         // This is what's setting the flag's value from the environment.
+        init: () => window.featureFlags.FEATURE_LOGIN,
+    },
};
...`}
</FileBlock>

If you wish to make the flag configurable through the admin `Experiments` page, you can also configure that here.

<FileBlock fileName='src/shared/featureFlags.ts'>
  {`...
export const flags: Record<Feature, IFeatureFlag> = {
    Login: {
        enabled: false,
        init: () => window.featureFlags.FEATURE_LOGIN,
        // The name that will appear on the 'Experiments' page
+        experimentName: 'Customer Single Sign-On',
        // Whether the value should be persisted between sessions or not.
+        persist: true,
    },
};
...`}
</FileBlock>

7. Add it to the webpack configuration.

This is the default value that will be used during development.

<FileBlock fileName='webpack.common.js'>
  {`...
const makeFeatureFlags = () => {
    const defaults = {
            ...
+        FEATURE_LOGIN: false,
            ...
    };
    ...
...`}
</FileBlock>

8. Add it to the jest configuration.

This is the default value that will be used in tests.

<FileBlock fileName='webpack.common.js'>
  {`...
module.exports = {
    ...
    featureFlags: {
            ...
+         FEATURE_LOGIN: true,
            ...
    },
    ...
...`}
</FileBlock>

9. Still there? Add it to the Circle CI config file.

<FileBlock fileName='.circleci/config.yml'>
  {`...
workflows:
  ...
  jobs:
    ...
    build-container:
      ...
      - run:
          name: Launch container for tests
          command: |
            set -x
            docker run --name happa-branch \\
              ...
              -e FEATURE_LOGIN=TRUE \\
              ...
...`}
</FileBlock>

10. Use it in code

<FileBlock fileName='YourFile.ts'>
  {`import * as featureFlags from 'shared/featureFlags';
...
    function doSomething() {
        if (featureFlags.flags.Login.enabled) {
            doSomeOtherThing();
        } else {
            doSomeAnotherThing();
        }
    }
...`}
</FileBlock>
